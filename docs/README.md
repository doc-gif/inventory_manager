# 🛠 開発ドキュメント (Architecture & Design Philosophy)

本ドキュメントでは、「くらしのストック」の開発におけるプロダクト概要、UI/UXの設計思想、および技術的な意思決定について解説します。

## 1. プロダクト概要 (Elevator Pitch)
**「買い忘れ」の根本原因である「在庫の把握漏れ」を解決する、在庫管理＋買い物リスト統合アプリ**

日用品や食料品の「買い忘れ」を防ぐためには、単なる買い物リストでは不十分であり、「現在何がどれだけ残っているか」を正確に把握する必要があります。しかし、在庫管理はユーザーにとって入力の手間（摩擦）が大きく、継続が難しいという課題があります。
本アプリは、この「入力コストの最小化」をUXの最重要課題と位置づけ、在庫管理と買い物リストをシームレスに統合することで、ユーザーの認知負荷を下げ、日常的な買い物をスマートにサポートします。

## 2. UI/UXの設計思想 (UI/UX Philosophy)
「機能の引き算」と「タップ数の最小化」による認知負荷の低減を徹底し、毎日ストレスなく使える「手触り」を実現しました。

* **機能の引き算 (Less is More):**
  開発者視点での過剰な機能追加を避け、「ユーザーが迷わず使いこなせる情報量」に制限。在庫管理と買い物リスト作成というコアバリューのみに焦点を絞りました。
* **マイクロインタラクションと導線設計:**
  アプリの継続利用の障壁となる「在庫の減少」「新規アイテムの追加」「リストの確認」といった主要アクションにおいて、画面遷移数とタップ数を極限まで削る設計を行いました。スーパーでの買い物中や日用品を使い切ったその瞬間に、ストレスなく片手で操作できる体験を実現しています。
* **ノイズレスなナビゲーションと視線誘導:**
  「リストの閲覧」と「商品の追加」のメンタルモデルを明確に分けるため、上部のセグメントコントロール（タブ）と右下のアクションボタン（FAB）に分離しました。

## 3. 技術選定の背景 (Technology Stack & Decision Making)
スマートフォンでの日常的な利用を想定し、要件と運用コストの観点から以下の構成を採用しました。

* **PWA (Progressive Web App):**
  * **配布コストの最適化:** 個人開発におけるApp Store / Google Playへのランニングコストや審査プロセスを排除。
  * **デバイス機能の要件クリア:** コア機能である「ローカルストレージ（データ保存）」および「カメラ機能（バーコード読み取り）」は、PWAのWeb API仕様で十分に要件を満たせると技術的に判断しました。
* **ホスティング環境 (Cloudflare Pages/Workers):**
  低遅延かつランニングコストを抑えるため、スケーラブルなエッジ環境を採用しています。
* **フレームワーク (Vite + React + TypeScript):**
  生成AI（LLM）を活用した高速な開発サイクルを実現するため、AIの学習データが最も豊富で、デファクトスタンダードであるReactエコシステムを意図的に選択しました。
* **状態管理 / スタイリング:** Zustand / Tailwind CSS

## 4. アーキテクチャと品質管理 (Architecture & Code Quality)
AIを用いたコーディングでは、コードの無秩序化（スパゲッティ化）が起きやすいという課題があります。これを防ぎ、高い可読性と低い複雑度（循環的複雑度や認知的複雑度など）を維持するため、AIの出力に対して厳密な設計上の制約（プロンプトエンジニアリング）を行いました。

* **関心の分離 (Separation of Concerns):**
  UI層とビジネスロジック層を明確に分離するよう指示し、コンポーネントの肥大化を防止しています。
* **レイヤードアーキテクチャの採用:**
  将来的なネイティブアプリへのリプレイスやチーム開発も見据え、高い拡張性を持つディレクトリ構造を採用しました。
  * **Presentation Layer (`/src/presentation`)**: UIコンポーネント（React）と、UIの状態管理を行うカスタムフックを配置。
  * **Application Layer (`/src/application`)**: グローバルな状態管理を行い、ユースケースを実現。
  * **Domain Layer (`/src/domain`)**: ビジネスルールや型定義、純粋な計算ロジックをカプセル化し、UIに依存しない設計としています。
